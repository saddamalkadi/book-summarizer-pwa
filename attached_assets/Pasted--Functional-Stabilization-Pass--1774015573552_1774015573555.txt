أريد الآن Functional Stabilization Pass فقط.

مهم جدًا:
- لا تغيّر التصميم الحالي.
- لا تغيّر الألوان.
- لا تغيّر التخطيط العام.
- لا تغيّر ترتيب الواجهة أو الـ navigation أو الـ hero أو البطاقات أو الهوية البصرية.
- لا تنفّذ أي redesign.
- اعتبر الشكل الحالي approved visually.
- المطلوب فقط: إصلاح الوظائف المعطلة مع الحفاظ الكامل على التصميم الحالي.

المشكلة الحالية:
- أزرار كثيرة في الواجهة لا تعمل كما يجب.
- تسجيل الدخول لا يعمل بالشكل المطلوب.
- أريد الحفاظ على نفس التصميم الحالي تمامًا، مع جعل التطبيق يعمل فعليًا بشكل كامل.

الهدف:
تحويل النسخة الحالية من “واجهة جميلة لكن بها أعطال وظيفية” إلى “واجهة جميلة وتعمل بالكامل”.

أولويات التنفيذ:
1) إصلاح جميع الأزرار غير العاملة
2) إصلاح تسجيل الدخول
3) التأكد أن الأزرار الرئيسية الثلاثة تعمل:
   - ابدأ محادثة
   - ارفع ملفات
   - مهمة جاهزة
4) التأكد أن أزرار التنقل تعمل
5) التأكد أن الأكورديون يعمل
6) التأكد أن أزرار الدردشة والإرسال والخيارات الثانوية تعمل
7) التأكد أن الأزرار داخل الملفات / OCR / التحويل / المشاريع / الإعدادات تعمل أو يتم تعطيلها بصريًا إذا لم تكن جاهزة
8) لا تترك أي زر يبدو قابلًا للنقر بينما هو بلا وظيفة

طريقة العمل المطلوبة:
المرحلة 1: Audit وظيفي فقط
- افحص كل الأزرار والعناصر القابلة للنقر في الواجهة الحالية
- أنشئ قائمة كاملة:
  - ما الذي يعمل
  - ما الذي لا يعمل
  - ما الذي يعمل جزئيًا
  - ما الذي يفشل بسبب JS
  - ما الذي يفشل بسبب selectors / event listeners / DOM timing / auth state / hidden overlays / disabled states
- افحص root cause الحقيقي
- لا تغيّر التصميم أثناء هذه المرحلة
- أنشئ:
  - docs/functional-audit-buttons-and-auth.md

المرحلة 2: إصلاح الأزرار
- أصلح كل event listeners المفقودة أو المكسورة
- أصلح أي مشاكل DOMContentLoaded / MutationObserver / dynamic rendering / re-render
- أصلح أي mismatch بين IDs / selectors / delegated events
- أصلح الأزرار الرئيسية أولًا ثم الثانوية
- إذا كان هناك زر لا يملك backend أو منطقًا جاهزًا، فلا تتركه مكسورًا:
  - إمّا فعّله
  - أو عطّله بوضوح مع tooltip/status "قريبًا"
- لا تترك أي زر صامت بلا استجابة

المرحلة 3: إصلاح تسجيل الدخول
- افحص تسجيل الدخول بالكامل على production flow
- افحص:
  - login button
  - email auth
  - session restore
  - logout
  - error states
  - loading states
  - auth UI after login
- أصلح root cause الحقيقي
- إذا كانت المشكلة من worker / gateway / token / callback / state sync / storage، أصلحها بالكامل
- بعد الإصلاح:
  - اختبر login end-to-end
  - وثّق السبب الجذري وما تم إصلاحه
- أنشئ:
  - docs/auth-functional-fix.md

المرحلة 4: Functional UX polish بدون تغيير التصميم
- فقط لتحسين الإحساس بالعمل:
  - أضف loading states واضحة
  - أضف disabled states واضحة
  - أضف error messages مفهومة
  - أضف success feedback
- لكن بدون أي تغيير بصري كبير أو redesign

المرحلة 5: Verification
- اختبر الواجهة الحالية كلها عمليًا
- أعطني جدولًا نهائيًا:
  - الزر
  - المكان
  - الحالة قبل
  - الحالة بعد
  - كيف تم الاختبار
- أنشئ:
  - docs/final-functional-verification.md

معايير القبول:
- لا يتغير التصميم الحالي
- لا تتغير الألوان أو الهوية
- الأزرار الرئيسية تعمل
- أزرار التنقل تعمل
- الأكورديون يعمل
- الدردشة تعمل
- تسجيل الدخول يعمل
- لا يوجد زر يبدو حيًا لكنه لا يفعل شيئًا
- كل الأعطال موثقة ومغلقة

مهم:
- لا تقل لي “تم” إلا بعد اختبار فعلي.
- اختبر على النسخة الحية وليس محليًا فقط إن أمكن.
- إذا تعذّر اختبار شيء على production بسبب الحاجة إلى حساب أو سرّ، اذكر ذلك بوضوح.
- ابدأ الآن بالمرحلة 1 فقط: functional audit للأزرار وتسجيل الدخول، ثم توقف وأعطني النتائج بالعربية.
