# v864 Prompt Engineering Mode

تم استبدال سلوك زر Focus بسلوك وظيفي جديد: **Prompt Engineering**.

## ما الذي تغيّر

- الزر العلوي نفسه (`focusModeBtn`) أصبح يمثل Prompt Engineering.
- عند التفعيل، النظام لا يرسل النص الخام مباشرة، بل يمر عبر helper:
  - `buildPromptEngineeringInput(originalText)`

## آلية العمل

1. يأخذ النص الأصلي كما كتبه المستخدم.
2. يعيد تنظيمه إلى صياغة أوضح (هدف/سياق/قيود/تعليمات تنفيذ).
3. يضيف النص الأصلي ضمن الحزمة لضمان حفظ النية.
4. يرسل النسخة المحسّنة إلى النموذج بدل النسخة الخام.

## ضمان النية والاستقرار

- النية الأصلية محفوظة لأن النص الأصلي مضمّن داخل الحزمة المحسنة.
- محتوى رسالة المستخدم الظاهر في الدردشة يبقى النص الأصلي.
- التعديل يقتصر على payload المرسل للنموذج عند تفعيل الوضع.
