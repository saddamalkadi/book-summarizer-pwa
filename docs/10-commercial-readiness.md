## 2. التسلسل المنطقي لاكتشاف المزايا

```
المستخدم الجديد يصل → شاشة تسجيل دخول بسيطة
       ↓
يدخل بالمجانية → شاشة رئيسية نظيفة
       ↓
يبدأ محادثة أو يرفع ملف → يكتشف القيمة الأساسية
       ↓
يستكشف الأدوات (canvas, transcription) → من الشريط الجانبي
       ↓
يطلب الترقية من داخل التطبيق
```

---

## 3. اللغة المستخدمة داخل المنتج

### جيدة ✅:
- "مساعدك الذكي" — شخصي وودود
- "ابدأ محادثة" — action-oriented واضح
- "مهمة جاهزة" — يوحي بالاحترافية
- "بحث عميق" / "تحليل ملف" — واضح ومباشر

### تحتاج مراجعة ⚠️:
- "KB" / "قاعدة المعرفة" — قد لا يفهمها المستخدم العادي → "المكتبة الذكية"
- "RAG / Retrieval" — مصطلح تقني → "بحث ذكي في ملفاتك"
- "OCR" — غير واضح → "استخراج نص من صورة"

## 5. ما يجعل المنتج يبدو "تجاريًا" لا "داخليًا"

### ✅ تم تطبيقه:
- Progressive disclosure: الإعدادات المتقدمة مخفية
- Home screen نظيف مع value proposition
- Navigation hierarchy واضح
- Font احترافي مدعوم بشكل كامل

### 🔲 للتطوير المستقبلي:
- Landing page خارجية تشرح المنصة قبل تسجيل الدخول
- صفحة pricing/plans داخلية واضحة
- onboarding tour تفاعلي للمستخدم الجديد
- Success metrics: عدد المحادثات، الملفات المحللة
