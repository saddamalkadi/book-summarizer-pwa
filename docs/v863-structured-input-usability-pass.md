# v863 Structured Input Usability Pass

الهدف من هذا التمرير: نقل التفاعل من "اكتب القيمة يدويًا" إلى "اختر من خيارات واضحة" أينما كانت القيم معروفة مسبقًا.

## قرارات UX الأساسية

- اعتماد `select` للحدود الرقمية الأكثر شيوعًا بدل الإدخال الحر.
- جعل خيار البوابة افتراضيًا موجّهًا عبر preset بدل كتابة URL كل مرة.
- الحفاظ على الإدخال الحر فقط عندما يكون مبررًا (مثل endpoint مخصص أو نموذج تضمين مخصص).

## تحسينات الدعم المنطقي

- تمت إضافة دالة `ensureSelectHasValue(...)` لضمان عدم فقدان القيم القديمة:
  - إذا كانت هناك قيمة محفوظة غير موجودة ضمن الخيارات القياسية، تضاف تلقائيًا كخيار "مخصص".
  - هذا يمنع كسر الإعدادات القديمة بعد التحويل إلى `select`.

## النتائج

- احتكاك أقل في الإعدادات اليومية.
- أخطاء أقل بسبب الكتابة اليدوية.
- وضوح أفضل لمسارات الضبط للمستخدم غير التقني.
