## 2. مشاكل شاشة تسجيل الدخول (Auth Gate)

### 2.1 حقول الدخول ليست داخل `<form>`
- **المشكلة:** حقل كلمة المرور يظهر كـ standalone مما يعطي تحذيرات المتصفح ويمنع إدارة كلمات المرور
- **التأثير:** المتصفح لا يعرض خيار "حفظ كلمة المرور" - تجربة مستخدم سيئة
- **الإصلاح:** تغليف الحقول في `<form onsubmit="...">` بدلًا من `<div>`

### 2.2 زر Google Sign-In لا يظهر في Replit
- **المشكلة:** نطاق Replit غير مسموح في Google OAuth
- **التأثير:** المستخدم يرى مساحة فارغة حيث المفروض يكون زر Google
- **التحسين المقترح:** رسالة واضحة تشرح ما الذي يجب فعله

### 2.3 رسالة الحالة الافتراضية طويلة جدًا
- **النص الحالي:** "سجّل الدخول ببريدك الشخصي لفتح الخطة المجانية، أو استخدم بريد الإدارة مع كلمة المرور من نفس الشاشة."
- **المشكلة:** نص طويل كـ placeholder للحالة، يبدو مكتظًا
- **الإصلاح المقترح:** تقليص النص وإضافة tooltip أو مساعد قصير

## 3. مشاكل التخطيط (Layout)

### 3.1 الصفحة الرئيسية مكتظة
- **المشكلة:** عند الدخول للتطبيق، توجد كثير من العناصر في الـ topbar والـ subtopbar
- **التأثير:** إرباك للمستخدم الجديد، لا يعرف من أين يبدأ
- **الإصلاح المقترح:** تبسيط الواجهة الأولى، إخفاء الخيارات الثانوية

### 3.2 شريط التنقل الجانبي
- **المشكلة:** يحتوي على 8+ عناصر تنقل بدون تجميع منطقي
- **التأثير:** المستخدم الجديد قد يتوه
- **الإصلاح المقترح:** تجميع العناصر في مجموعات (رئيسية / إضافية)

### 3.3 Responsive Design
- **المشكلة:** على الشاشات الصغيرة (< 768px)، التخطيط يستخدم grid ثنائي الأعمدة
- **التأثير:** قد يكون الـ sidebar ضيقًا جدًا أو الـ chat صغيرًا
- **الحالة:** يوجد collapse logic لكنه يحتاج تحسينًا

## 4. مشاكل تجربة المستخدم (UX)

### 4.1 غياب Onboarding للمستخدم الجديد
- **المشكلة:** لا توجد شاشة ترحيب أو دليل أول استخدام
- **التأثير:** المستخدم الجديد لا يفهم كيفية إضافة API key أو البدء
- **الإصلاح المقترح:** wizard بسيط أو tooltip موجّه عند أول زيارة

### 4.2 حالة الدردشة الفارغة
- **المشكلة:** عند فتح محادثة جديدة، المساحة فارغة
- **الإصلاح المقترح:** رسالة ترحيب واضحة مع اقتراحات للبدء

### 4.3 رسائل الخطأ عند فشل AI
- **المشكلة:** بعض أخطاء AI تظهر كنص تقني مطوّل
- **الإصلاح المقترح:** تحسين رسائل الخطأ لتكون أكثر ودية ووضوحًا

### 4.4 زر الميكروفون (Voice)
- **المشكلة:** يظهر للمستخدم حتى عندما يكون الصوت غير متاح
- **التأثير:** المستخدم يضغط ولا يحدث شيء أو يحدث خطأ
- **الإصلاح المقترح:** إخفاء أو تعطيل الزر مع رسالة واضحة

## 5. مشاكل RTL والعربية

### 5.1 بعض العناصر المبنية ديناميكيًا
- **الحالة:** معظم النصوص العربية صحيحة
- **المشكلة الطفيفة:** بعض الرسائل الإنجليزية قد تظهر بتنسيق LTR داخل سياق RTL
- **الأولوية:** منخفضة

### 5.2 خطوط النظام
- **الحالة:** يعتمد على `system-ui` وخطوط النظام
- **على Android/iOS:** الخطوط العربية تبدو جيدة
- **على Windows:** تبدو جيدة مع "Segoe UI"
- **توصية:** إضافة fallback لخط عربي مخصص (مثل Noto Sans Arabic) لتجربة موحّدة

## 6. مشاكل الأداء

### 6.1 تحميل مكتبات CDN
- **المشكلة:** 6 مكتبات تُحمَّل من CDN بشكل متسلسل (بعضها sync)
- **التأثير:** قد يُبطئ وقت التحميل الأولي
- **الإصلاح المقترح:** جعل كل المكتبات `async` أو `defer` حيث أمكن

### 6.2 `localStorage` كمخزن رئيسي
- **المشكلة:** كل البيانات في `localStorage` — محدودية 5-10 MB
- **للمستخدمين الكثيفين:** قد يمتلئ المخزن
- **الحالة:** KB يستخدم IndexedDB (أفضل)
